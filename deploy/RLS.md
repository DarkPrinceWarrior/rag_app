# RLS — второй контур изоляции документов (ТЗ §4.7.1)

Основная изоляция по владельцу — на уровне приложения (owner-фильтр в
`retrieve()`, поиске, чате, папках; проверено). RLS — **второй контур**
(defense-in-depth) на уровне БД: даже если код забудет owner-фильтр, СУБД не
отдаст чужие строки.

## Что уже сделано (миграция 0020)

- `ENABLE ROW LEVEL SECURITY` + политики `*_owner` на `documents`, `folders`,
  `chunks`, `segments`, `document_translations`, `segment_versions`.
- Политика: строка видна, если `app.is_admin='on'` (service/admin) **или**
  владелец строки = `app.user_id` **или** владелец NULL (dev-документы).
- GUC `app.user_id` / `app.is_admin` выставляется в каждой транзакции хуком
  `after_begin` (`src/rag_app/db/rls.py`) из contextvar-принципала. Аутентифи-
  цированный API-запрос ставит принципала пользователя; воркер/миграции/старт —
  service (bypass). Fail-open: ошибка хука → service-режим, запрос не падает.

## Статус: контур АКТИВЕН в проде

Роли БД на сервере (`.env.api.local` / `.env.worker.local`):

| Роль | BYPASSRLS | Владелец таблиц | Используется | RLS |
|------|-----------|-----------------|--------------|-----|
| `rag` | да (super) | да | миграции (`alembic`) | обходит (DDL) |
| **`rag_api`** | **нет** | нет | **API** | **применяется** |
| `rag_worker` | да | нет | воркер (ARQ) | обходит (service) |

Поскольку API ходит под `rag_api` (без BYPASSRLS, не владелец), политики 0020
**реально фильтруют** строки по GUC `app.user_id`/`app.is_admin`. Проверено
параллельно под двумя учётками: `engineer` (user) НЕ видит документы `ruslan` ни
в списке, ни в поиске, ни в чате; обращение к чужому документу/сегментам/переводу
→ 404. Воркер под `rag_worker` (BYPASSRLS) проходит все пайплайны.

Особо: `list_segments` (`/documents/{id}/segments`) исторически не имел
app-level owner-проверки — её закрывает активный RLS (и дополнительно добавлена
явная проверка владельца для defense-in-depth паритета с соседними роутами).

## Замечание

Не запускать API под ролью `rag` или `rag_worker` (обе BYPASSRLS) — это отключит
второй контур. Боевая роль API — строго `rag_api` (без BYPASSRLS).
